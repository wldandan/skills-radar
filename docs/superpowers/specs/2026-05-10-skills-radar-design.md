# Skills Radar 项目设计规范

> **版本**: v1.0.0
> **创建日期**: 2026-05-10
> **状态**: 待用户审核

---

## 1. 项目概述

### 1.1 项目背景

Skills 正在成为 Agent 落地的关键载体，围绕 Skills 的技术（生成、评测、优化、管理）正在快速演进。然而目前缺乏一个系统性追踪 Skills 技术发展的平台，从业者难以把握技术全貌和趋势。

### 1.2 项目目标

- **Skills 技术雷达**：按照 Skills 相关技术发展，收录最新技术，按生命周期分类，追踪演进阶段
- **Top 10 Skills 深度分析**：采集自 clawhub、skills.sh、Claude Skill，**以 skill-insight 为分析引擎**，多维评测 10 个热门 Skills，展示 skill-insight 的召回追踪、执行流图、效果评测、智能归因、自优化能力
- **skill-insight 最佳实践**：Skills Radar 成为 skill-insight 能力的具象化案例库，证明 skill-insight 在真实场景中的分析价值
- **独立品牌运营**：Skills Radar 作为独立内容品牌，skill-insight 作为核心分析工具被自然、深入地展示

### 1.3 目标用户

| 用户群体 | 核心需求 |
|---------|---------|
| Agent 开发者 | 了解 Skills 技术选型、找到合适的 Skill 分析方法 |
| 企业决策者 | 把握 Skills 行业趋势、评估技术成熟度 |
| 研究者/学术 | 追踪前沿技术、了解技术演进路径 |

## 2. 内容架构

### 2.1 Part 1: Skills 技术雷达

#### 2.1.1 分类维度（基于 Skill-insight 生命周期）

| 生命周期阶段 | 技术领域 | 示例技术 |
|-------------|---------|---------|
| **Skill 生成** | 文档解析、语义聚合、去冗余、模式抽取 | Trace2Skill、D2Skill、SkillX |
| **Skill 召回** | 意图识别、路由策略、召回算法 | SkillRouter、SkillOrchestra、Graph of Skills |
| **Skill 执行** | 渐进式披露、执行追踪、状态管控 | SkVM、渐进式加载、执行流图 |
| **Skill 评测** | 多维评测、准确率、召回率、归因分析 | SkillsBench、SkillProbe、Skill-insight |
| **Skill 优化** | 自优化、迭代优化、效果提升 | SkillForge、Skill-insight optimizer |
| **Skill 管理** | 版本管理、权限管控、生命周期管理 | SkillNet、AgentSkillOS |

#### 2.1.2 展示形式：组合视图

1. **Radar 图**（蜘蛛网图）
   - 多维技术维度：技术成熟度、创新性、落地程度、生态活跃度
   - 每种生命周期技术的综合评分

2. **时间线**（Timeline）
   - 按技术出现时间排列
   - 展示技术演进历程和关键里程碑

3. **分类目录**（Taxonomy）
   - 按生命周期六阶段分类
   - 每阶段下按技术类型细分

#### 2.1.3 技术成熟度评估

| 成熟度等级 | 定义 | 标签色 |
|-----------|------|-------|
| 探索期 | 概念验证阶段，无成熟产品 | 🔴 红 |
| 成长期 | 有原型实现，开始落地应用 | 🟡 黄 |
| 成熟期 | 已有成熟产品，大规模应用 | 🟢 绿 |
| 衰退期 | 已过时或被新技术取代 | ⚫ 灰 |

### 2.2 Part 2: Top 10 Skills 分析（核心：展示 Skill-insight 分析引擎能力）

#### 2.2.1 核心目标

**Skills Radar 作为 skill-insight 的最佳实践展示窗口**：通过分析热门 Skills，将 skill-insight 的多维评测能力、技术归因能力、自优化能力具象化呈现，让用户直观感受到 skill-insight 能解决什么问题、如何使用。

#### 2.2.2 数据来源

| 平台 | 采集方式 | 说明 |
|-----|---------|------|
| Claude Skill Market | 官方 API / 页面抓取 | Anthropic 官方 Skills |
| clawhub | 抓取 / API | 社区 Skills 聚合平台 |
| skills.sh | 抓取 / API | 另一个 Skills 聚合平台 |

#### 2.2.3 分析框架（基于 skill-insight 全链路能力）

```
用户任务输入
    ↓
意图识别 + Skill 召回（Skill Router）
    ↓
Skill 执行追踪（执行流图 vs 预期流程）
    ↓
多维效果数据采集（准确率/召回率/Token/时延）
    ↓
智能归因分析（Skill 设计问题 vs 模型能力问题）
    ↓
针对性优化建议 + 再评测闭环
```

#### 2.2.4 分析维度（必须全部覆盖 skill-insight 核心能力）

**1. 召回分析**（Skill Router / 召回率评测）
- 意图识别准确率
- Top-5 召回率
- 误召率分析
- 展示方式：召回路径可视化

**2. 执行追踪**（Skill-insight 执行流图）
- 实际执行路径 vs 预期路径对比
- 跳步/冗余/偏差标识
- 关键节点耗时分解
- 展示方式：执行流图对比

**3. 效果评测**（多维指标）
- 准确率（任务完成度）
- Skill 召回率
- Token 消耗（CPSR）
- 执行时延
- 投入产出比
- 展示方式：雷达图/柱状图

**4. 归因分析**（智能归因引擎）
- Skill 设计缺陷定位
- 模型能力边界识别
- 优化优先级建议
- 展示方式：归因树状图

**5. 优化建议**（自优化能力展示）
- 基于归因的修复建议
- 优化前后效果对比
- 展示方式：对比图

## 4. 品牌与 Skill-insight 关系策略

### 4.1 Skills Radar 定位

- **Slogan**: "追踪 Skills 技术，让 Agent 能力进化有迹可循"
- **视觉识别**: 独立 Logo、独立配色方案
- **内容调性**: 专业、前沿、中立、深度

### 4.2 Skill-insight 关系定位

**Skills Radar 是 skill-insight 的最佳实践展示窗口**：

- **分析引擎**：Top 10 分析以 skill-insight 为唯一下线分析工具，所有评测数据来源于 skill-insight
- **能力展示**：每篇分析报告都要明确展示 skill-insight 的哪个能力被使用、解决了什么问题
- **案例库价值**：Skills Radar 积累的 Top 10 分析案例，成为 skill-insight 的官方案例库
- **自然推广**：通过高质量分析让读者感受到 skill-insight 的价值，而非硬广

### 4.3 与 tech-blog 的协同

- 作为 "Agent Skills洞察与实践" 专栏的姐妹篇
- 互补定位：专栏侧重深度文章，Radar 侧重技术追踪 + skill-insight 实践
- 可以在 tech-blog 文章中引用 Radar 数据和分析结果
