## 基本信息

**一句话描述**: **并行多 sub-agent 编队**从轨迹池蒸馏 Skill —— 整体归纳，避免逐条轨迹过拟合。35B 模型 +57.65pp 超越 122B。

**来源**:

**链接**:

## 核心技术

### 并行多 Agent 分析

不是一条条轨迹顺序处理，而是**同时派出一支 sub-agent 编队**并行分析整个轨迹池，分两组：

- **成功分析师** —— "这些做对的案例有什么共同点？"
- **错误分析师** —— "这些做错的案例卡在哪？"

每组独立提出 Skill 补丁建议。

### 层级合并 + 程序化冲突检测

所有补丁不是简单拼接，而是通过**层级归纳推理**合并为一个无冲突的 Skill 目录。

**例**：分析师 A 建议"遇到 timeout 时重试 3 次"，分析师 B 建议"切换备用 API" → 合并器识别冲突，选择支持证据更多的方案，或组合为"先重试 3 次，仍失败则切换"。

### 关键数字

- **35B 模型 + Trace2Skill Skill 在 WikiTableQuestions 上超越 122B agent +57.65pp** —— 小模型 + 好 Skill 可打败大模型

## 技术细节

### 主要能力

- 并行轨迹分析，规模化
- 程序化冲突检测 + 格式验证
- 跨模型迁移有效

### 局限性

- 需要**大量并行执行轨迹**作为输入，冷启动场景受限
- 合并阶段在"信号部分相关或存在领域偏差"时的行为尚不清楚
- 蒸馏出的 Skill 可解释性较差 —— "人很难读懂 LLM 写给自己的 Skill"
