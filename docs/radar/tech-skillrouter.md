## 核心技术

### 两阶段检索 - 重排

1. **双编码器检索（粗排）** —— Qwen3-Emb-0.6B 把查询与全量 Skill 向量化，检索 Top-20 候选
2. **交叉编码器重排（细排）** —— SR-Rank-0.6B 通过交叉注意力做精细化相关性打分

### 推翻"标题+简介够用"的设计假设

在 ~80K Skill 池上实验，**隐藏 Skill body 后路由准确率暴跌 31-44 个百分点**。

> 两本书封面都写"数据可视化指南"，但一本教 Python Matplotlib，另一本教 Excel —— 不翻开内容根本分不清。

### 紧凑高效

- 1.2B 总参数量 vs 16B Qwen3 基线（Hit@1 0.641 vs 0.637）
- **推理速度快 13×**
- 可部署在消费级硬件上

### 端到端验证

不止"路由准了"就完事 —— 在 Kimi-K2.5、GLM-5、Claude Sonnet/Opus 4.6 四个主流 Agent 上验证，路由准确率提升**确实转化为最终任务成功率提升**。

### 实验结果

| 指标 | 结果 |
|---|---|
| 检索阶段 Hit@1 | 65.4%（超越 13× 参数量的 8B 基线） |
| 端到端 Hit@1 | **74.0%**（比最优零样本 8B 基线 +6pp） |
| 重排序器价值 | 修复 12.7% 的检索失败案例 |

---

## 技术细节

### 局限性

- 74% Hit@1 意味着仍有 26% 首选路由错误
- 只做单次"选哪个 Skill"决策，**不涉及多 Skill 动态编排**
- 识别出"相似 Skill"后没有方案做自动合并
