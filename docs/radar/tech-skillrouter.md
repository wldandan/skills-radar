## 核心技术

### 核心原理

SkillRouter 提出**两阶段检索-重排流水线**，核心发现是**技能实现文本（Skill Body）是匹配精度的核心支撑**：

1. **双编码器检索（粗排）**：采用 Qwen3-Emb-0.6B 将用户查询与全量技能库进行向量嵌入，检索 Top-20 候选
2. **交叉编码器重排（细排）**：采用 SR-Rank-0.6B 通过交叉注意力机制对每个候选进行精细化相关性得分排序

### 核心创新

- **全字段感知**：完整纳入技能实现文本，而非仅依赖元数据
- **极致轻量化**：1.2B 总参数量，无需依赖超大模型
- **端到端优化**：两阶段架构可针对技能路由场景进行端到端微调

## 技术细节

### 主要能力

- 解决上下文容量瓶颈问题（8万技能库场景）
- 解决技能匹配精度问题（功能重叠的技能区分）
- 跨模型迁移能力

### 适用场景

- 大规模技能库（10万+级别）的精准匹配
- 端侧 Agent 的技能路由
- 多模型、多框架环境下的技能选择

### 实验结果

| 指标 | 结果 |
|------|------|
| 检索阶段 Hit@1 | 65.4%（超越13倍参数量的8B基线） |
| 端到端 Hit@1 | 74.0%（比最优零样本8B基线提升6个百分点） |
| 重排序器价值 | 修复12.7%的检索失败案例 |
