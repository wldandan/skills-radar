# Skills 技术成熟度评估模型

> **版本**: v2.0 · **更新日期**: 2026-05-12

---

## 一、评估维度

| 维度 | 描述 | 权重 |
|---|---|:---:|
| **技术成熟度** | 从论文实验到生产部署的演进程度 | 30% |
| **创新性** | 相对现有路径的原创性和方法论突破 | 25% |
| **落地程度** | 在真实产品 / 项目 / 企业场景中的应用广度 | 25% |
| **生态活跃度** | GitHub Star · 引用 · 贡献者 · 商业生态 | 20% |

**综合评分公式**：`overall = 0.30 × maturity + 0.25 × innovation + 0.25 × adoption + 0.20 × ecosystem`

---

## 二、成熟度等级定义

| 等级 | 标签 | 综合评分 | 典型特征 |
|---|:---:|:---:|---|
| **探索期** | 🔴 | < 0.40 | 痛点已被研究但缺少有效方案；仅论文或早期原型 |
| **成长期** | 🟡 | 0.40 – 0.60 | 有原型实现，开始落地应用；多个开源项目 |
| **成熟期** | 🟢 | ≥ 0.60 | 有可工程化部署的方案，大规模实际应用 |

## 三、4 维评分细则

### 3.1 技术成熟度

| 分数 | 阶段 |
|:---:|---|
| 0.00 - 0.45 | 论文阶段，无开源实现 |
| 0.45 - 0.65 | 论文 + 开源实现 |
| 0.65 - 0.80 | 论文 + 开源 + 工业验证 |
| 0.80 - 1.00 | 标准 / 大规模生产部署 |

### 3.2 创新性

| 分数 | 说明 |
|:---:|---|
| 0.00 - 0.45 | 渐进改进，无原创性突破 |
| 0.45 - 0.65 | 有一定方法论创新 |
| 0.65 - 0.80 | 显著创新，被业界引用 |
| 0.80 - 1.00 | 范式级突破，引领方向 |

### 3.3 落地程度

| 分数 | 说明 |
|:---:|---|
| 0.00 - 0.35 | 仅学术实验或闭源内部使用 |
| 0.35 - 0.55 | 小范围试点、社区早期采用 |
| 0.55 - 0.75 | 多个实际产品采用 |
| 0.75 - 1.00 | 官方标准 / 行业广泛部署 |

### 3.4 生态活跃度

| 分数 | 说明 |
|:---:|---|
| 0.00 - 0.40 | 无社区或论文阶段，少量更新 |
| 0.40 - 0.60 | 中等社区、稳定维护 |
| 0.60 - 0.80 | 活跃社区、高频更新 |
| 0.80 - 1.00 | 重量级生态（如 4K+ ★、官方背书） |

## 四、categorySummary 的特殊计算

每个 category 的综合分**不机械平均所有产品**，而是综合考虑：

1. 该 category 下产品数量及典型代表的评分
2. **业界核心痛点的解决程度**（参考 Skill-insight 竞品分析中的 🟢🟡🔴⚪ 证据分类）
3. **是否存在结构性空白**（如管理阶段的"防御工具为零"会显著拉低评分）

| 阶段 | 综合评分 | 等级 |
|---|:---:|:---:|
| Skill 召回 | **0.65** | 🟢 成熟期 |
| Skill 优化 | **0.58** | 🟡 成长期 |
| Skill 生成 | **0.55** | 🟡 成长期 |
| Skill 评测 | **0.45** | 🟡 成长期 |
| Skill 执行 | **0.42** | 🔴 探索期 |
| Skill 管理 | **0.32** | 🔴 探索期 |
