# Skills 技术成熟度评估模型

> **版本**: v1.0.0
> **创建日期**: 2026-05-10
> **Story**: R-003
> **状态**: 初稿

---

## 一、评估维度

技术成熟度从四个维度评估：

| 维度 | 描述 | 权重 |
|------|------|------|
| **技术成熟度** | 概念验证 → 原型 → 成熟产品的演进程度 | 30% |
| **创新性** | 技术的原创性和前沿程度 | 25% |
| **落地程度** | 在实际产品/项目中的应用广度 | 25% |
| **生态活跃度** | 社区贡献者数量、GitHub Star、更新频率 | 20% |

---

## 二、成熟度等级定义

| 等级 | 标签 | 定义 | 典型特征 |
|------|------|------|---------|
| **探索期** | 🔴 红 | 概念验证阶段，无成熟产品 | 论文发表、早期原型、无商业落地 |
| **成长期** | 🟡 黄 | 有原型实现，开始落地应用 | 1-2 个开源项目、少量实际使用、早期采用者 |
| **成熟期** | 🟢 绿 | 已有成熟产品，大规模应用 | 多个成熟开源项目、大规模实际应用、稳定迭代 |
| **衰退期** | ⚫ 灰 | 已过时或被新技术取代 | 维护减少、无新特性、社区转移 |

---

## 三、评分标准

### 3.1 技术成熟度评分标准

| 分数 | 阶段 | 说明 |
|------|------|------|
| 0.0-0.25 | 🔴 探索期 | 仅论文或概念，无实现 |
| 0.25-0.50 | 🟡 成长期 | 有原型，实验性使用 |
| 0.50-0.75 | 🟡 成长期 | 成熟开源项目，实际应用案例 |
| 0.75-1.0 | 🟢 成熟期 | 稳定版本、大规模应用、文档完善 |

### 3.2 创新性评分标准

| 分数 | 说明 |
|------|------|
| 0.0-0.25 | 增量改进，无原创性突破 |
| 0.25-0.50 | 有一定的技术创新 |
| 0.50-0.75 | 显著创新，被引用较多 |
| 0.75-1.0 | 开创性工作，引领方向 |

### 3.3 落地程度评分标准

| 分数 | 说明 |
|------|------|
| 0.0-0.25 | 仅实验室测试 |
| 0.25-0.50 | 小范围试点使用 |
| 0.50-0.75 | 多个实际产品采用 |
| 0.75-1.0 | 行业标准，大规模部署 |

### 3.4 生态活跃度评分标准

| 分数 | 说明 |
|------|------|
| 0.0-0.25 | 无社区、很少更新 |
| 0.25-0.50 | 小社区、低频更新 |
| 0.50-0.75 | 中等社区、稳定维护 |
| 0.75-1.0 | 活跃社区、高频更新、大量贡献者 |

---

## 四、综合评分计算

**综合评分 = 技术成熟度 × 0.3 + 创新性 × 0.25 + 落地程度 × 0.25 + 生态活跃度 × 0.2**

### 评分区间

| 综合评分区间 | 成熟度等级 |
|-------------|-----------|
| 0.0 - 0.35 | 🔴 探索期 |
| 0.35 - 0.60 | 🟡 成长期 |
| 0.60 - 1.0 | 🟢 成熟期 |
| 衰退指标 | ⚫ 衰退期（维护减少 > 1 年） |
