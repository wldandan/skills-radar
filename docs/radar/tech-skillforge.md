## 核心技术

### Domain-Contextualized Skill Creator

从历史工单挖掘两类信息：

1. **工具挖掘** —— 识别专家在工单中高频使用的诊断工具，提取其 schema
2. **知识抽取** —— 从内部文档和工单引用的参考资料中提取领域规则

**例**：分析 1000 个"ECS 无法启动"工单后发现：专家 90% 时间先跑 `ecs-diag` → 再查 VPC 安全组 → 最后看系统日志 —— 这个诊断路径被编码为 Skill。

### 三阶段自优化流水线（关键创新）

| 阶段 | 角色 |
|---|---|
| **Failure Analyzer** | 分析执行失败 |
| **Skill Diagnostician** | 将失败追溯到具体 Skill 缺陷（如"Skill 没有告诉 Agent 在操作前检查实例状态"） |
| **Skill Optimizer** | 重写 Skill 消除根因 |

关键在于**直接将失败归因到 Skill** —— 这是归因分离的初步方案。

### 真实生产验证

5 个真实云技术支持场景，使用 **1,883 条匿名化生产工单 + 3,737 个任务**。

## 技术细节

### 局限性

- 仅在云技术支持场景验证，**通用性未知**
- 代码和数据未开源（企业隐私）
- 聚焦**单 Skill** 优化，**不涉及多 Skill 协同**
- Diagnostician 只做"是否 Skill 的问题"这一层归因，不区分 Skill 内容 / 模型能力 / 路由错误
