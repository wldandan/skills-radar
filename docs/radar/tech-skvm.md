## 核心技术

### 核心原理

SkVM 把传统编译器思路搬到 Skills 领域，让 Skills 能像普通代码一样在不同模型、不同环境下高效稳定运行：

- **AOT 编译器（Ahead-of-Time）**：安装 Skills 时生成适配当前环境的 Skills 变体
- **运行时系统**：执行时做 JIT 优化和资源调度

### 核心创新

**AOT 编译三阶段**：
1. **基于能力的编译**：定义26个基础能力，量化 Skills 与目标环境的能力差距，差得不多就"补偿"，差得太多就"替换"
2. **环境绑定**：提前到编译阶段检查依赖，缺的话生成安装脚本
3. **并发提取**：挖掘 Skills 里隐藏的并行性（数据级、指令级、线程级）

**JIT 优化**：
- **自适应重编译**：记录执行情况，反复失败时自动重新优化
- **代码固化**：将重复代码模板转成可调用函数

## 技术细节

### 主要能力

- 能力补偿与适配变换
- 依赖预检测与安装
- 并行性提取与调度
- 代码固化加速

### 实验结果

| 指标 | 效果 |
|------|------|
| 任务成功率 | 平均提升15.3%，小模型最多25% |
| Token 消耗 | 最多降低40% |
| 速度提升 | 并发提取最快3.2倍，代码固化最快50倍 |
| 框架一致性 | 差距从13分缩小到5分 |

### 局限性

- 目前仍在研究阶段
- 对框架的并行能力支持有要求
