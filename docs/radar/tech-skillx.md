## 基本信息

**一句话描述**: 把 Skill 显式分为**策略层 / 功能层 / 原子层三层粒度**，强模型经验向弱模型蒸馏。

**链接**:

## 核心技术

### 三层 Skill 设计

| 层 | 含义 | 例子 |
|---|---|---|
| **策略层** | 高层规划 | "先认证再操作" |
| **功能层** | 具体步骤 | "用 OAuth2 获取 Token" |
| **原子层** | 最小操作单元 | "发送 POST 请求" |

> 像乐高积木 —— 底座不同但上面的结构件通用。上层 Skill 跨 Agent 复用，底层 Skill 适配不同环境。

### 强模型蒸馏给弱模型

GPT-4 级别模型积累的 Skill 经验可被 GPT-3.5 级别模型直接消费。AppWorld / τ²-Bench 上弱模型获得显著提升。

### 解决"团队各自重复发明轮子"

公司 10 个团队各自教 Agent 调用内部 API → SkillX 让一个团队的经验沉淀成可复用三层 Skill 给其他团队用。

---

## 技术细节

### 主要能力

- 三层 Skill 粒度自动划分
- 跨模型蒸馏
- 跨团队 Skill 复用

### 局限性

- 跨环境迁移需要**底层工具集相对稳定** —— 如果两个环境的 API 完全不同，上层 Skill 也难复用
- **没有自动化工具帮使用者判断"当前 Skill 是太抽象了还是太具体了"** —— 三层粒度是设计准则而非自动决策
- 学术阶段，未产品化
