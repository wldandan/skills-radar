# SkillClaw

> **分类**: Skill 优化 / Skill 管理 | **成熟度**: 🟡 成长期 | **综合评分**: 0.55

## 基本信息

**一句话描述**: **后台守护进程**静默聚合多用户的所有 Agent 轨迹 → 识别跨用户重复模式 → 进化 Skill → 同步回全用户。

**来源**:
- AMAP-ML（阿里高德地图）
- 论文 + 开源

**链接**:

## 核心技术

### 三阶段管线

| 阶段 | 行为 |
|---|---|
| **① 结构化会话收集** | 用户 prompt → Agent 动作 → 中间反馈 → 最终答案，所有用户轨迹汇入同一证据池 |
| **② Agentic Evolver 识别模式** | 自主推理引擎分析所有轨迹，识别**跨用户反复出现的行为模式**（如 5 个工程师独立发现"先备份 → 分块 → 合并"三步流程 → 提炼为 Skill） |
| **③ 共享仓库同步** | 进化后的 Skill 存入共享仓库，自动同步到所有用户的所有 Agent 实例 |

### 关键设计洞察

> 单个用户很少产生足够信号区分"可泛化改进"和"特定场景补丁"，但聚合多用户互补视角后，哪些模式真正通用就变得清晰。

原生集成 Hermes / OpenClaw / QwenPaw 以及任何 OpenAI 兼容 API。

## 技术细节

### 局限性

- **仅在 Qwen3-Max 上验证**
- **隐私和冲突解决未详述** —— 当不同用户经验矛盾时（"日志流式读取更快" vs "全量读取更稳定"），SkillClaw 强制合并为统一 Skill，**缺乏分支或覆盖机制**
- **企业保密场景顾虑** —— 多用户轨迹汇聚意味着 A 的操作模式对 B 间接可见
