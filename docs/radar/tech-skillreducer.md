## 核心技术

### 核心原理

针对 Skill 普遍存在的**技能膨胀（Skill Bloat）**问题，SkillReducer 提出两阶段优化框架：

**阶段1：路由层优化**
- 模拟预言的 Delta 调试：将原始描述拆分为独立语义子句，递归分割与测试
- 真实环境验证：确保压缩后描述仍能正确触发技能

**阶段2：技能主体重构（渐进式披露）**
- 分类驱动的内容拆分：核心规则常驻，补充内容按需加载
- 分类型定制化压缩：示例/模板去重，背景精准摘要
- 双重质量门+自校正反馈循环

### 核心创新

- **技能膨胀实证研究**：对55315个公开技能的大规模分析
- **少即是多效应**：移除非必要内容减少了LLM的上下文干扰
- **忠实性验证**：确保压缩后内容完整保留核心操作概念

## 技术细节

### 主要能力

- 路由描述压缩（平均48%压缩率）
- 技能主体压缩（平均39%压缩率）
- 功能保留验证（86%技能表现不降低）

### 实验结果

| 指标 | 效果 |
|------|------|
| 路由描述压缩率 | 48%（已有完整描述的技能56.5%） |
| 技能主体压缩率 | 39%（平均节省约1000 token） |
| 端到端节省 | 26.8% 输入 Token |
| 功能质量提升 | 2.8%（部分技能反而提升） |
| 路由保留率 | 100%（非过时技能） |

### 局限性

- 4.7% 真实压缩失败率
- 技能过时导致的问题无法通过压缩解决
