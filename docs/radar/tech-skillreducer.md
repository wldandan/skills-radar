# SkillReducer

> **分类**: Skill 优化 | **成熟度**: 🟡 成长期 | **综合评分**: 0.55

---

## 基本信息

**一句话描述**: 大规模实证（**55,315 个 Skill**）揭示 Skill 生态系统性冗余 —— description 压缩 48% + body 压缩 39%，**功能质量反升 2.8%**（less-is-more 效应）。

**来源**:
- 学术研究 - 港科大等

**链接**:
- 论文：[arXiv 2603.29919](https://arxiv.org/abs/2603.29919)

## 核心技术

### 大规模实证发现

| 发现 | 数据 |
|---|---|
| **没写 description 的 Skill** | 26.4% |
| **body 中"非可操作"内容** | 60%+ |
| **reference 文件注入量** | 数万 Token |

### Stage 1 — 路由层优化（Delta Debugging）

用对抗性方法找 description 的"最小充分子集"。

> 你有一串 50 个关键词的 description，SkillReducer 逐个删除测试"Agent 还能正确找到这个 Skill 吗？"，最终压缩到只留最关键的 10 个词。压缩率 **48%**。

### Stage 2 — Body 重构（渐进披露）

Skill body 分层：
- **核心规则**（永远加载） —— 如"组件必须是纯函数"
- **背景知识 + 示例**（按需加载） —— 如"关于 React Hooks 的历史"

Agent 默认只看"首页"，需要时才翻"文件柜"。压缩率 **39%**。

### "Less-is-more" 效应

压缩后功能质量反而 **+2.8%** —— 干扰 Agent 注意力的噪音少了。在四个家族五个模型上迁移有效。

---

## 技术细节

### 局限性

- 聚焦**静态压缩**（让已有 Skill 更精简），**不涉及 Skill 内容进化或新 Skill 发现**
- 压缩是通用的，不针对特定失败模式做定向优化
- 仍是学术论文阶段，**未产品化**
- 未做"基于当前任务的动态按需加载"
