## 基本信息

**一句话描述**: 把数十万 Skill 组织成 **Capability Tree + DAG 编排**的"操作系统级"基础设施。

## 核心技术

### Capability Tree 递归分类

把数十万 Skill 组织成一棵能力树：节点是能力类别，叶子是具体 Skill。用户提任务时，系统从树根递归定位到最相关 Skill 子集。

对比线性搜索全部 Skill，**树状检索在 200 - 200K 规模上都接近理想准确率**。

### DAG 编排（核心创新）

识别任务中的 Skill 依赖关系，构建 **有向无环图**：
- 能并行的步骤并行执行（如"抓取数据" + "准备 PPT 模板"）
- 必须串行的按序执行（如"清洗数据"必须等"抓取"完成）

DAG 执行**显著优于**简单逐个调用。

### 基准

30 个 artifact-rich 任务基准，覆盖：
- 数据计算 / 文档创建 / 动态视频 / 视觉设计 / Web 交互

---

## 技术细节

### 局限性

- DAG 编排假设 Skill 依赖关系**可在执行前静态确定** —— 动态依赖场景（"第二步选择取决于第一步结果"）未覆盖
- 不涉及 Skill 进化或安全治理
- 仅学术验证，未产品化
