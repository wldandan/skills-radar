## 基本信息

**一句话描述**: 第一个把 **Skill 本身**作为评测对象的标准化基准 —— 7,308 条轨迹 / 84 任务 / 11 领域，揭示"加 Skill 反而更差"等反共识结论。

## 核心技术

### 三条件对比设计

同一任务在三种条件下跑 —— 像药物试验的安慰剂对照组：

1. 不用 Skill
2. 用精心编写的 Skill
3. 用 Agent 自己生成的 Skill

### 关键发现（不是解决方案，是问题揭示）

| 发现 | 数据 |
|---|---|
| **精心编写的 Skill 显著有效** | 平均提升 +16.2pp |
| **但领域差异巨大** | 软件工程仅 +4.5pp，医疗 +51.9pp（差了 10×） |
| **自生成 Skill 平均无增益** | 学生自己写复习笔记帮不了自己 |
| **部分场景 Skill 有害** | 84 任务中 **16 个负增益** —— 加了反而更差 |
| **聚焦优于全面** | 2-3 模块精简 Skill 优于长篇文档 |

### 关键意义

SkillsBench 不解决问题，但**第一次量化**了 Skill 生态中最重要的几个反共识事实，是后续所有评测和优化工作的起点。

---

## 技术细节

### 局限性

- 只做 **pass/fail 结果评测**，**不追踪 Agent 是否实际遵从了 Skill 流程**
- 无法解释 16 个负增益案例的根因（"Agent 忽略了 Skill？Skill 有缺陷？路由选错？"）
- 84 个确定性验证器都是手工编写的，覆盖 11 个领域 —— 但 SkillNet 上有 40 万个 Skill 横跨数百领域，绝大多数没有评测基础设施
